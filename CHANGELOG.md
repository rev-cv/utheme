# Changelog

## 2.7.0 — 2026-07-21
- Компиляция SCSS переведена на встроенный dart-sass внутри плагина — больше не требует Docker/Node-контейнера, пересборка идёт по кнопкам Save/Random и в фоновом режиме отслеживания правок, с журналом последних 100 попыток компиляции
- Добавлен плагин Sonar: единая настройка классификации сайта и реального/обфусцированного имени темы на стандартной странице настроек WordPress, плюс скрытый эндпойнт `/sonar-ping` для программного опознания сайтов пайплайна во всём парке
- Cookie-плашка получила два новых визуальных варианта (push-banner, edge-bar) с автоподбором цветовой схемы и поддержкой обфускации классов
- Удалены неиспользуемые legacy-скрипты деплоя (`deploy-scripts/`)

## 2.6.0 — 2026-07-20
- Вывод пайплайна и destroy.py переведён на Rich: прогресс-бары и живые таблицы вместо построчного print, интерактивное подтверждение удаления сайта стрелками. Пайплайн консолидирован с 13 до 9 фаз
- Микроразметка Schema.org расширена: FAQPage для FAQ-блоков, CollectionPage для страниц-листингов, корректный author у Article
- Внутренние shorthand-ссылки резолвятся через кэшированную карту слагов вместо запроса к БД на каждую ссылку
- Восстановлена активация плагина стилей темы и GEO-плагина при деплое; исправлены утечки временных папок при обработке картинок

## 2.5.0 — 2026-07-02
- Alt-тексты из SEO-метаданных теперь автоматически записываются в медиатеку WordPress при деплое
- Формат изображений в манифесте расширен до объекта `{filename, alt}` для передачи alt-текста по пайплайну

## 2.4.2 — 2026-07-01
- Модуль TOC и geo-виджетов разделён на два независимых файла: `mod-toc.php` и `mod-geo-integration.php`
- Geo-виджеты всегда встают после h1; ручное позиционирование через шорткоды убрано
- Временно отключены активация плагина стилей темы и настройка GEO при деплое

## 2.4.1 — 2026-06-29
- Исправлена навигация TOC с кириллическими якорями (болгарский и др.)
- Деплой ускорен: импорт медиа без лишних ресайзов, замена плейсхолдеров батчем
- Исправлена ошибка компиляции шаблона provision.sh при наличии `${#...}` bash-синтаксиса

## 2.4.0 — 2026-06-29
- SVG-изображения проходят через пайплайн без конвертации в WebP (compress + wp_html)
- `card-grid` конвертер обрабатывает `<span>`-дочерние элементы как параграфы
- Alt-текст миниатюр в виджете «ещё статьи» берётся из медиатеки WordPress
- Разрешение директорий в extract_meta приоритизирует `index.html`/`index.md`

## 2.3.0 — 2026-06-15
- Независимое ручное позиционирование TOC и рекламных блоков через шорткоды `[toc_position]`, `[geo_position]`, `[sport_position]`
- Авто-вставка рекламы теперь учитывает обёртку `<figure>` и корректно ставит блок перед ней
- h2 внутри инжектируемых блоков скрыты стилями; бумажный фон статей отключён

## 2.2.0 — 2026-06-11
- Markdown теперь нативный формат: прямой конвертер Markdown→Gutenberg без промежуточного HTML
- Сканирование и переименование изображений поддерживает .md-файлы (headimg из frontmatter + inline-изображения)
- Брендинговые файлы (logo, favicon) принимаются из подпапки IMAGES/; рабочие папки (NEW, PROMTS, IMAGES) молча пропускаются при обходе структуры

## 2.1.1 — 2026-06-09
